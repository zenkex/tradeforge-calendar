### 项目名称

**P&L Calendar** - 本地交易盈亏日历记录工具

### 项目目标

开发一个美观、专业、纯本地运行的交易盈亏日历网页，专门用于外汇/期货交易者记录每日 P&L。完全仿照 Lucid Trading 的 Calendar 风格设计，强调清晰、直观、交易氛围。

### 核心需求

- **纯前端实现**：单个 HTML 文件内嵌所有 CSS 和 JS
- **数据 100% 本地存储**：使用 `localStorage`，不发送任何数据到服务器
- **月度视图**：默认显示当前月，支持上/下月切换，一键回到今天
- **周日作为每周起始**：日历第一列为周日，最后一列为周六
- **周末自动识别**：周六、周日显示为休市状态（不可输入，特殊样式）
- **日期颜色规则**：
  - **绿色**：当日盈利（P&L > 0）
  - **红色**：当日亏损（P&L < 0）
  - **灰色**：未交易或 P&L = 0
- **手动输入**：点击工作日日期弹出 Modal 输入当日盈亏金额（支持正负数）和备注

### UI/UX 要求

1. **整体风格**：
   - 深色交易主题（zinc-950 / zinc-900 背景），支持一键切换亮色主题
   - 英文使用 Space Grotesk 字体，中文使用 Noto Sans SC 字体
   - P&L 金额数字使用 JetBrains Mono 等宽字体
   - 卡片式布局，圆角 `rounded-xl`，适当阴影
   - 字体从 Google Fonts 加载，设置 `font-display: swap`

2. **主题切换**：
   - 暗色主题为默认，亮色主题通过 `.light-theme` CSS class 覆盖所有颜色
   - 亮色模式：背景 `#f1f5f9`，卡片 `#ffffff`，文字 `#1e293b` ~ `#94a3b8`
   - 切换按钮位于页面右上角，显示 ☀️ / 🌙 图标
   - 主题偏好持久化到 `localStorage`（key: `themePreference`）

3. **日历布局**：
   - 顶部：年月切换按钮（`←` `→`）+ "今天"快捷按钮 + "清空本月"按钮
   - 表头：日、一、二、三、四、五、六（周日和周六表头文字淡化）+ "每周盈亏"列
   - 日期格子：日期数字固定灰色（`text-zinc-400`，不随盈亏变色）+ 当日 P&L 金额
   - 格子背景半透明着色：盈利 `bg-emerald-500/10`，亏损 `bg-red-500/10`，无交易 `bg-zinc-800/30`
   - 周一至周五格子边框为 2px（`border-2`），周末格子无边框
   - 今日日期格子右上角显示琥珀色小圆点（`bg-amber-400`），附加发光圆环效果

4. **交互**：
   - 点击工作日日期打开 Modal 输入 P&L
   - Modal 中显示日期、金额输入框（前缀 `$` 符号，`type="text" inputmode="decimal"`，带输入过滤）、备注输入框、保存/取消/删除按钮
   - Modal 打开时金额输入框自动聚焦，光标定位到末尾
   - 点击周末日期无反应
   - 键盘快捷键：`Esc` 关闭任意 Modal，`Enter` 保存/确认，`←` `→` 切换月份

5. **每周盈亏列**：
   - 日历最右侧独立列（第 8 列），表头显示"每周盈亏"
   - 每行显示该周在当前月份内的日期 P&L 总和
   - 盈利绿色、亏损红色、无数据留空
   - 字号和边框与日期格子一致（`border-2`）

6. **金额显示格式**：
   - 所有 P&L 金额保留两位小数（如 `$1,250.75`、`-$450.00`）
   - 正数不显示 `+` 号，只显示 `$` 前缀（如 `$500.00`）
   - 负数显示 `-` 号和 `$` 前缀（如 `-$500.00`）
   - 日历格子内金额因空间有限，大数值使用 k/M 缩写（如 `$1.25k`、`-$2.50k`）
   - 统计面板使用完整格式（带千位分隔符）

### 技术实现规格

#### 数据结构

所有账户数据存储在 `localStorage` 的 `pnlCalendars` 键下：

```json
{
  "默认账户": {
    "2026-05-01": { "pnl": 1250.75, "note": "日内突破交易" },
    "2026-05-02": { "pnl": -450.00, "note": "止损调整不当" }
  },
  "实盘账户": {
    "2026-05-01": { "pnl": 3200.00, "note": "" }
  }
}
```

- `pnlLastAccount` — 记录上次使用的账户名，刷新后自动恢复
- `themePreference` — 记录主题偏好（`dark` / `light`）
- 首次使用时自动迁移旧版 `pnlCalendar` 单键数据到多账户结构

#### 多账户管理

- 自定义下拉菜单（非原生 `<select>`）：按钮显示当前账户名 + `▼` 箭头，点击展开菜单
- 菜单中每个账户行：点击名称切换账户、点击 ✏️ 编辑名称、点击 🗑️ 删除
- 菜单底部「＋ 新建账户」选项
- 新建/编辑/删除账户均使用自定义 Modal（非浏览器原生弹窗），样式与编辑 PnL Modal 一致
- 账户名长度限制 24 字符，不可重名、不可为空
- 删除至少保留一个账户，删除当前账户后自动切换到剩余第一个账户
- 切换账户后所有界面（日历、统计、图表）自动刷新为该账户数据
- 上次使用的账户名持久化，刷新页面后恢复

#### CSS 颜色规范

- 盈利绿：`text-emerald-400` / `bg-emerald-500/10` / `border-emerald-500/20`
- 亏损红：`text-red-400` / `bg-red-500/10` / `border-red-500/20`
- 休市/无交易：`bg-zinc-800/30` / `text-zinc-400`
- 周末：`bg-zinc-900/30` / `text-zinc-600`（无边框，cursor: default）
- 今日标记：`bg-amber-400`
- 统计占位符/空数据：`text-zinc-500`
- 危险操作按钮：`bg-red-500/10` / `text-red-400` / `border-2 border-red-500/20`

#### 资金曲线图表（Canvas 2D 自绘）

- 累计盈亏折线图，从 X=0、Y=$0 起点开始
- 线段按 Y 值正负分段着色（绿色 / 红色），零轴跨越时精确计算交点并拆分
- 数据点颜色：>0 绿、<0 红、=0 灰
- 填充区：盈利区绿色半透明、亏损区红色半透明
- Y=0 基准线加粗加亮显示
- Y 轴标签在左侧，X 轴标签显示日期数字
- 鼠标悬浮最近数据点时显示 tooltip 和虚线指示线
- 仅显示至当月最后一个有 P&L 数据的日期；整月无数据时显示"本月无数据"
- 主题切换时图表颜色自动适配
- 窗口 resize 时 debounce 重绘

#### 核心函数

- `fmtDate(y, m, d)` — 格式化为 `YYYY-MM-DD` 字符串
- `fmtPnL(value, compact)` — 格式化 P&L 金额，compact 模式用于格子内显示
- `getDaysInMonth(y, m)` — 获取指定月份天数
- `getFirstDayOffset(y, m)` — 计算当月 1 日是周几（0=周日）
- `isWeekend(y, m, d)` — 判断是否周六或周日
- `loadData()` / `saveData()` — localStorage 多账户读写
- `switchAccount(name)` — 切换当前账户并刷新界面
- `getEntry(dateStr)` / `setEntry(dateStr, pnl, note)` / `deleteEntry(dateStr)` — 当前账户数据 CRUD
- `calcTradeStats(entries)` — 计算胜率、平均盈亏、盈亏比、最佳/最差日、连续盈亏等
- `renderCalendar()` — 渲染日历网格 + 每周盈亏列
- `renderStatsCards()` — 渲染本月/今年/全部总盈亏卡片
- `renderAdvancedStats()` — 渲染高级统计面板（无数据时全部显示灰色占位符）
- `drawChart()` — Canvas 2D 绘制累计盈亏折线图（主题自适应）
- `openModal(dateStr)` / `closeModal()` / `saveModalEntry()` / `deleteModalEntry()` — PnL 编辑 Modal
- `exportData()` / `importData()` — JSON 导入导出（当前账户范围）
- `clearCurrentMonth()` / `doClearMonth()` — 清空当前月数据（自定义确认 Modal）
- `getTheme()` / `toggleTheme()` / `applyTheme()` / `initTheme()` — 主题管理
- `filterPnlInput(el)` — 金额输入框实时过滤（仅允许数字、负号、小数点）
